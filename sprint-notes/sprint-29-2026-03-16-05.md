## 구현 내역

### 1. 퀘스트 시스템 코어 (Frontend + Research)
- `include/quest.h` — 신규 헤더:
  - QuestState enum: NOT_STARTED(0), ACTIVE(1), COMPLETE(2), REWARDED(3) — 2비트 인코딩
  - QuestType enum: COLLECT, HUNT, DELIVER
  - QuestData struct: id, type, giver info, target info, reward info
- `src/core/quest.c` — 신규 구현:
  - quest_flags 비트필드 기반 상태 관리 (4 퀘스트/바이트, bytes 0-5)
  - 고블린 킬 카운터: quest_flags[6]
  - 배달 플래그: quest_flags[7]
  - 퀘스트 타입별 완료 조건 체크
  - 보상 지급 (수집 퀘스트는 아이템 소모)

### 2. 퀘스트 3개 (Research 설계)
| 퀘스트 | 타입 | 의뢰 NPC | 목표 | 보상 |
|--------|------|----------|------|------|
| 0 - Herb Collection | COLLECT | Old Sage (map 0) | Potion 3개 | Potion 1개 + 50G |
| 1 - Goblin Hunt | HUNT | Guard (map 1) | Goblin 3마리 | Iron Blade + 30G |
| 2 - Delivery | DELIVER | Merchant (map 0) → Oasis Trader (map 2) | 대화 | 100G |

### 3. NPC 대화 통합 (Frontend)
- `src/core/game.c` — 퀘스트 연동:
  - NPC 상호작용 시 퀘스트 상태별 대사 분기
  - NOT_STARTED → 퀘스트 제안 + A 버튼으로 수락
  - ACTIVE → 리마인더 대사
  - COMPLETE → 완료 대사 + 보상 지급
  - REWARDED → 감사 대사
  - 전투 승리 시 `quest_on_enemy_defeated()` 호출로 킬 카운트 추적

### 4. 테스트 (QA)
- `tests/test_quest.c` — 10개 테스트 함수, 75 assertions:
  - 상태 get/set (2비트 조작)
  - 상태 전이 (NOT_STARTED → ACTIVE → COMPLETE → REWARDED)
  - 3개 퀘스트 타입별 완료 조건 검증
  - 보상 지급 검증
  - quest_flags 세이브/로드 라운드트립
  - NPC 대화 라우팅
